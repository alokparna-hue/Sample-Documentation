== Purpose
This document standardizes access provision for all eligible associates and contains the steps necessary to provide, revoke or reassign access to ABC's internal applications.

== Scope
This document is applicable for IT admins and support engineers. This does not cover access provison for external websites or applications.

== Definitions
The following table describes the terms used in the document.
|===
| *Term* | *Description* |

| IAM
| Identity and Access Management refers to the process of managing user identities and system-access controls.

|Requestor
|This refers to the associate who has requested access to any system or application.

|Approver
|This refers to the admin (associate) who approves and grants access to the requested application.
|===

== Setup procedure
* *Clone* the repository
* *Install* dependencies
* *Configure* environment variables
* *Start* the application

== Verification
*Confirm* that the application starts without any errors.

NOTE: _Ensure all prerequisites are installed before beginning._

WARNING: _Do not commit environment files to version control._
