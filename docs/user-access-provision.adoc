= User access provision
:toc:
:toclevels: 2
:sectnums:

== Purpose
This standard operating procedure (SOP) standardizes access provision for all eligible associates and contains the steps necessary to provide, revoke or reassign access to ABC's internal applications.

== Scope
This document is applicable for IT admins and support engineers. This does not cover access provison for external websites or applications.

== Definitions
The following table describes the terms used in the document.
|===
| *Term* | *Description*

| IAM
| Identity and Access Management refers to the process of managing user identities and system-access controls.

|Requestor
|This refers to the associate who has requested access to any system or application.

|Approver
|This refers to the designated approver (associate) who approves and grants access to the requested application.
|===

== Roles and Responsibilities
The following table lists the roles and responsibilities of the associates involved in access provisioning.
|===
| *Role* | *Responsibility* | *Restrictions*

|Requestor
|*Requests* access to necessary applications and *provides* justification for access.
|*Cannot* approve any requests.

|Approver
|*Approves* the access requests and grants access. _For example_, a manager or an admin.
|*Cannot* approve requests without any valid business justification.

|IT administrator
|*Authorizes*, *reviews*, *documents*, and *manages* the user access provision processes.
|*Cannot* grant access without approvals or deviate from the SOP.
|===
